# 说明

本计划用于说明奇绩前端展示的所有要求和与约束，以及相关的开发计划，最终目的是生成符合奇绩风格的前端展示页面，用于展示当前"Vibe for 奇绩"所搭建和评测的Claude Code Skills。
网页部署在github网页端，所以为静态网页，请你使用符合要求的程序配置。

# 前端设计要求

## 功能

1. 标题说明 Vibe for 奇绩，核心是使用Claude Code Skill助力奇绩的工作，更高质量，更长期地赋能奇绩校友成长
2. 需要网页本体的github仓库链接，在提交到github仓库后我会提供链接
3. 主体展示部分需要展示我们已经评测和搭建的Skill，分为两类（在前端上需要体现设计）
    a. 我们评测的Skill
        i. 提供Skill原仓库的github链接
        ii. 提供详细的使用说明文档链接按钮（背后为飞书链接）
        iii. 提供必要的配置说明，比如依赖哪些必要的配置，需要提前安装哪些软件（在前端以类似tag的方式展示出来）
        iv. 提供简短的一句话简介，此外在旁边附上评分，五分制，用星星数量展示
    b. 我们搭建的Skill：配置同上，但是要更加醒目
4. 在右侧边栏提供问卷悬浮按钮（不要过于醒目，但是一定要能够看见并点开），问卷标题为"参与奇绩Skill评测"，同样为飞书问卷的链接，后续我会提供。
5. 整体展示已经评测过和已经自行设计的Skill数量在前端，展示总数
6. 提供筛选的方法，可以筛选对应领域的Skill，可以使用

## 展示风格

1. 主题色：
    a. 奇绩蓝：088CFF，主体颜色
    b. 纯白色
    在品牌传播中，奇绩蓝为主体颜色，需大面积使用，以蓝白简约风格为主。
2. 整体风格
    a. 蓝白简约风格，但是请你设计，使用frontend skill
    b. 直观展示Skill
    c. 合理使用动画效果，要与Claude Skills使用过程相契合

## 验收约束

- 禁止使用utf-8码自带的表情和icon，如果需要对应的icon，可以跟我说明
